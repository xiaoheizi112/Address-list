# 通讯录管理系统

## 1.项目背景

通讯录管理系统是一个基础的个人信息管理工具，旨在帮助用户方便地存储、查找和管理联系人信息。在数字化时代，高效管理个人社交和工作联系人信息变得尤为重要，本项目通过简洁直观的界面和功能，为用户提供了一个轻量级的通讯录解决方案。

## 2.技术栈

- **编程语言**：C语言
- **数据结构**：单向链表（用于动态存储联系人信息）
- **标准库**：
  - stdio.h - 输入输出操作
  - stdlib.h - 内存分配和释放
  - string.h - 字符串处理函数
  - bool.h - 布尔类型支持
  - unistd.h - 系统调用（如sleep）
  - ctype.h - 字符类型判断和转换

## 3.实现功能

1. **添加联系人**：录入联系人姓名和电话号码
2. **删除联系人**：根据姓名删除指定联系人
3. **显示所有联系人**：展示通讯录中的所有联系人信息
4. **修改联系人**：更新指定联系人的姓名和电话号码
5. **查找联系人**：根据姓名搜索联系人信息
6. **退出系统**：安全退出并释放所有动态分配的内存

## 4.达到目的

本项目旨在提供一个简洁、高效、易用的通讯录管理解决方案，帮助用户：
- 系统化管理个人或工作联系人信息
- 快速查找和访问联系人信息
- 便捷地更新和维护联系人数据
- 避免信息丢失和混乱

## 5.应用方向

- **个人日常使用**：管理个人社交、家庭和朋友的联系方式
- **工作环境**：维护同事、客户和合作伙伴的联系信息
- **学习示例**：作为C语言编程和数据结构（链表）的学习参考案例
- **小型应用集成**：可作为其他小型应用中的联系人管理模块

## 6.项目亮点

1. **动态数据存储**：采用链表结构实现动态内存管理，根据需要分配和释放内存
2. **完善的输入验证**：对电话号码进行数字验证，确保数据正确性
3. **用户友好界面**：简洁明了的菜单系统和操作提示，提供良好的交互体验
4. **健壮的错误处理**：处理各种边界情况和错误输入，提高系统稳定性
5. **高效的内存管理**：程序结束时自动释放所有动态分配的内存，避免内存泄漏
6. **代码结构清晰**：模块化设计，函数分工明确，便于维护和扩展

## 7.如何运行该项目

### 前提条件

- 需要安装C语言编译器，如GCC、Clang或Visual Studio的C编译器

### 编译和运行步骤

#### 在终端下：
1. 使用GCC编译器编译源代码：
   gcc main.c addresslist.c -o addresslist
2. 运行生成的可执行文件：
   ./addresslist

### 使用说明

运行程序后，您将看到主菜单，可以通过输入数字1-6选择相应的功能：

1. **添加联系人**：输入联系人姓名和电话号码
2. **删除联系人**：输入要删除的联系人姓名
3. **显示所有联系人**：查看所有已保存的联系人信息
4. **修改联系人**：输入要修改的联系人姓名，然后输入新的姓名和电话号码
5. **查找联系人**：输入要查找的联系人姓名
6. **退出系统**：结束程序运行

## 8. 编写项目时遇到的问题及解决方法

### 问题一：为什么用fgets接收输入信息而不是scanf？
答：
1. fgets函数可以安全地读取包含空格的字符串，而scanf在读取字符串时会在遇到空格时停止，导致无法正确获取完整的输入。
2. fgets在遇到错误或文件结束时返回NULL，可以用于检测输入异常，每次读取都从新行开始，状态更可控。
3. 联系人姓名常包含空格、中间名等，fgets比scanf更适合处理这类复杂字符串。

### 问题二：为什么使用有头节点的单链表用于保存联系人信息？
答：
1. 头节点的存在可以简化链表的操作，如在头节点后插入新节点或删除头节点后的节点。
2. 头节点的指针域可以指向链表的第一个节点，而不是直接指向数据域，这在遍历链表时更方便。
3. 有头节点的单链表在删除第一个节点时，不需要特殊处理，直接将头节点的指针域指向第二个节点即可。

### 问题三：为什么要单独创建尾指针用于保存链表的最后一个节点？
答：
1. 尾指针的作用是指向链表的最后一个节点，这在添加新节点时非常方便。
2. 利用尾指针可以直接将新节点连接到链表的末尾，而不需要遍历整个链表。
3. 尾指针的存在可以提高添加联系人的效率，尤其是在链表比较长时。

### 问题四：在销毁链表函数当中，为什么需要使用临时变量temp来释放curr节点？
答：
1. 释放内存时，需要先将当前节点curr的指针域赋值给临时变量temp，然后释放curr节点的内存。
2. 这样可以确保在释放内存后，curr节点的指针域不再指向无效内存，避免悬空指针问题。
3. 临时变量temp的存在可以帮助我们安全地释放当前节点curr，而不会影响到链表的其他部分。
